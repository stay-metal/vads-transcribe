# config/it_ai_terms.txt — курируемый словарь канонических EN IT/AI-терминов для ПРАЙМИНГА L2.
#
# НАЗНАЧЕНИЕ: эти канонические написания подмешиваются в whisper initial_prompt «второго мнения»
# (whisper_asr._build_context). Цель — чтобы L2 выдавал английские IT/AI-термины ПРАВИЛЬНОЙ
# латиницей, а fusion слил их поверх ЛАТИНСКИХ манглов GigaAM.
#
# ЭТО НЕ ПРАВИЛА-ЗАМЕНЫ. В отличие от config/glossary.json (alias→canon, который переписывает текст),
# этот файл лишь подсказка написания для L2. Поэтому:
#   • I1 не затрагивается: verbatim-кириллица не меняется, fusion остаётся латиница-only;
#   • риска кириллических манглов/гомографов (рак→RAG) НЕТ — здесь нет ключей-алиасов;
#   • НЕ требует корпусных доказательств (в отличие от glossary-алиасов).
# Кириллические транслитерации (GigaAM пишет термин кириллицей) этим НЕ чинятся — только латинские
# манглы; кириллица→EN остаётся ручным, корпус-доказательным glossary-алиасом (precision-first).
#
# Формат: одна каноническая форма на строку; пустые строки и `#`-комментарии игнорируются; регистр
# СОХРАНЯЕТСЯ (это канон). Дубликаты с канонами glossary.json безвредны (whisper-подсказка).
#
# ⚠️ ПОРЯДОК ВАЖЕН. whisper initial_prompt обрезается с НАЧАЛА (whisper_asr._prompt_text оставляет
# хвост ~900 символов), а этот словарь идёт в контексте ПЕРВЫМ → выживает ХВОСТ файла. Поэтому
# порядок: СВЕРХУ — частые/простые термины (whisper и так пишет их верно, не жалко обрезать),
# СНИЗУ — самые ценные/редкие/неоднозначные (ядро AI/ML + имена продуктов/библиотек).
